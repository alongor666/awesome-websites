# 贡献指南

感谢您对常用网址集合项目的关注！我们欢迎任何形式的贡献。

## 🎯 如何贡献

### 1. 添加新的网址

如果您想要添加新的网址，请遵循以下步骤：

1. **Fork** 本仓库到您的GitHub账户
2. **Clone** 您的仓库到本地：
   ```bash
   git clone https://github.com/您的用户名/awesome-websites.git
   cd awesome-websites
   ```
3. 创建新的分支：
   ```bash
   git checkout -b add-new-website
   ```
4. 在相应的分类文件中添加网址，格式如下：
   ```markdown
   - [网站名称](https://www.example.com/) - 网站简要说明
   ```
5. **Commit** 您的更改：
   ```bash
   git add .
   git commit -m "添加：网站名称到分类文件"
   ```
6. **Push** 到您的仓库：
   ```bash
   git push origin add-new-website
   ```
7. 创建 **Pull Request**

### 2. 网站要求

请确保您提交的网站符合以下要求：

- ✅ 内容质量高，对用户有价值
- ✅ 网站能正常访问
- ✅ 没有恶意内容
- ✅ 不重复已有收录
- ✅ 包含准确的描述
- ✅ 归类到正确的分类

### 3. 提交格式

请使用以下格式添加网站：

```markdown
- [网站名称](https://www.example.com/) - 简洁准确的网站描述（20-30字为宜）
```

### 4. 分类说明

目前包含以下分类：

- **开发工具** - 编程相关的工具和资源
- **学习资源** - 技术学习、教程和课程
- **设计资源** - UI/UX设计相关资源
- **效率工具** - 提升工作效率的工具
- **娱乐休闲** - 娱乐和生活类网站
- **搜索引擎** - 各类搜索引擎
- **社区论坛** - 技术和兴趣社区

## 📝 提交规范

### Commit Message 格式

请使用清晰的 Commit Message，例如：

- `添加：XXX网站到开发工具分类`
- `更新：完善网站描述`
- `修复：移除无效链接`
- `重构：重新组织网站分类`

### Pull Request 规范

PR 标题请使用：
- `添加：XXX网站`
- `更新：优化分类结构`
- `修复：错误链接`

PR 描述请说明：
- 添加了哪些网站
- 为什么这些网站有价值
- 做了哪些改进
